## The Equation Chain

The recurve bow is not linear. It is interior, recurve, two-directional. Expressed as a magnitude inequality chain:

```
AF > BE ≥ CD ≥ (L-N)(R-N) < G = Vt
```

| Term | Meaning |
|------|---------|
| **AF** | Failure vector pair (bow tips A and F, declared at manifest: A=0, F=1) |
| **BE** | Abstraction pair (left and right abstractions, mirror-paired around G) |
| **CD** | Trough/aim pair (current context C and macro aim D) |
| **(L-N)(R-N)** | NEW: the L-N and R-N layer — addition, the equation's growth direction |
| **G = Vt** | Golden wavefront equals velocity vector — the firing arrow |

Magnitude reads:

- `AF` is largest (the declared boundary span)
- `BE ≥ CD ≥ (L-N)(R-N)` — descending nested magnitudes through the inner anchors
- `(L-N)(R-N) < G` — G strictly exceeds the (L-N)(R-N) floor
- The wavefront punches up out of the floor

This is the **structural inequality form** of the recurve bow. The original linear `A→B→C→G→D→E→F` chain in `recurve_bow.rana` is the **traversal sequence**. The two forms describe the same architecture from different angles.
